# 彩票验奖 Mac App · 设计交接文档（Design Brief）

> 交付对象：UI/UX 设计师
> 版本：v1（基于现有功能实现整理）
> 平台：macOS 桌面应用（SwiftUI 原生）
> 文档目的：说明产品定位、信息架构、每个页面的内容/状态/交互，便于设计师据此产出视觉与交互稿。本文不限定具体视觉风格，留给设计师发挥；但需覆盖文中列出的所有页面、元素与状态。

---

## 1. 产品概述

一款给个人彩民使用的「彩票验奖与记录」桌面工具，覆盖中国两种主流彩票：

- **双色球（ssq）** — 福彩
- **大乐透（dlt）** — 体彩

核心流程：**拍/选彩票照片 → AI 识别号码 → 用户核对确认 → 拉取官方/自建开奖结果 → 自动判奖 → 记录并出统计图表。**

设计关键词建议：清晰、可信赖、轻量、数据可读。涉及金额与中奖结果，需要让用户一眼看清「中了没 / 中多少 / 赚还是亏」。

### 目标用户
- 个人彩民，长期购彩，希望集中管理纸质彩票、自动对奖、回顾盈亏。
- 非专业用户，操作要简单；但也需要能手动纠错（识别可能出错、开奖源可能不一致）。

### 设计目标
1. 验奖流程顺畅，从选图到出结果尽量少步骤、状态清晰。
2. 号码、金额、中奖等级等关键数据高可读。
3. 「盈亏」类财务信息有概览、有明细、有图表。
4. 体现「可信赖」：开奖结果来源透明、可追溯、可手动修正。

## 3. 信息架构 / 导航

左侧侧边栏共 6 个一级入口（顺序即当前顺序），每项一个图标 + 文案：

| 入口 | 图标（SF Symbol）| 作用 |
|---|---|---|
| 首页 | house | 概览：核心数据 + 最近彩票 |
| 验奖 | doc.viewfinder | 上传图片识别并验奖（主操作）|
| 彩票列表 | list.bullet.rectangle | 所有彩票，点进详情 |
| 验奖结果总览 | checkmark.seal | 表格形式汇总各票最新结果 |

二级页面（非侧边栏，从一级页面进入）：
- **彩票详情**（从「彩票列表」进入）
- **开奖版本管理弹窗**（从「彩票详情」进入，模态 Sheet）

> 建议设计师额外考虑：当前默认进入「首页」。可评估是否将「验奖」作为更突出的主操作（例如全局悬浮按钮或首页大号 CTA）。

## 4. 领域知识（设计前必读）

号码以「号码球」形式展示，区分前后区，用颜色区分：

| 彩种 | 前区（红球）| 后区（蓝球）|
|---|---|---|
| 双色球 ssq | 6 个，范围 1–33 | 1 个，范围 1–16 |
| 大乐透 dlt | 5 个，范围 1–35 | 2 个，范围 1–12 |

- 前区当前用**红色**、后区用**蓝色**圆形号码球（两位数补零，如「07」）。
- 号码球有「命中 / 未命中」两种状态：命中=实心彩色 + 白字；未命中=灰底。设计师需为命中态做出明显视觉强调（验奖结果可视化的核心）。
- 当前仅支持「单式」（一注固定号码）。**复式 / 胆拖为「开发中」**，界面里有按钮但置灰禁用——设计时请预留这两种玩法未来的展示空间。

**中奖等级**（用于结果展示，金额分两类）：
- *固定奖*：等级对应固定金额（如双色球三等奖 ¥3000、大乐透三等奖 ¥10000 等）。
- *浮动奖*：一、二等奖金额不固定，需从开奖数据中读取；若缺失，显示「金额以官方为准」。

**数据源（开奖结果来源）**，需在界面上作为标签透明展示：

| 标识 | 显示名 | 适用彩种 |
|---|---|---|
| officialSporttery | 官方·体彩 | 仅大乐透 |
| officialCWL | 官方·福彩 | 仅双色球 |
| webService | Web 服务 | 自建服务，任意彩种 |
| manual | 手动录入 | 任意彩种（永远可用的兜底）|

## 5. 页面逐页规格

> 每页给出：用途 → 内容元素 → 关键状态 → 交互。设计师需为每页覆盖「空 / 加载中 / 错误」等状态。

### 5.1 首页（概览 Dashboard）

**用途**：打开即看到的总览页。

**内容元素**
1. 标题「概览」。
2. **核心数据卡片**（4 张，横向一排）：
   - 累计投入（¥）
   - 累计中奖（¥）
   - 净盈亏（¥，可正可负 → 建议正绿负红或其它语义色）
   - 中奖率（百分比）
3. **最近彩票**列表（最多 5 条）：每条显示 `[彩种] 第 X 期`，右侧显示最近一次中奖金额（无则显示「—」）。
4. 底部一段灰色提示文案，引导用户去「验奖 / 统计 / 详情」。

**状态**：无任何彩票时 → 需要空状态设计（引导去验奖）。

---

### 5.2 验奖（核心流程页）

**用途**：本 App 最重要的操作页。上传图片 → 识别 → 核对 → 验奖出结果。当前为单页表单式（分区块），设计师可重构为更有引导性的步骤式流程。

**内容元素 / 流程分区**

1. **彩票图片区**
   - 图片预览（选图后显示，最大高度约 180pt）。
   - 「选择图片…」按钮（打开系统文件选择器）。
   - 「识别」按钮（未选图或处理中时禁用）。

2. **确认（可编辑）区** —— 识别结果回填到此，用户可改：
   - 彩种选择（双色球 / 大乐透）。
   - 期数（文本输入）。
   - 前区/红球（空格分隔的号码输入）。
   - 后区/蓝球（空格分隔的号码输入）。
   - 数据源选择（按所选彩种动态过滤可用源）。
   - 「复式/胆拖（开发中）」按钮——置灰禁用。

3. **状态提示文字**：识别/验奖过程与结果都在此呈现（普通提示=次要色；以「错误」开头=红色）。

4. **「验奖」按钮**（处理中禁用）。

**关键状态**（务必都设计）
- 初始空白态。
- 已选图、未识别。
- 识别中（loading）。
- 识别完成、待核对。
- 识别失败 / 未配置模型（错误态，引导去设置）。
- 验奖中（loading）。
- 验奖结果：**中奖**（显示等级 + 金额，或「金额以官方为准」）/ **未中奖** / **该期未开奖或不存在** / 号码校验错误。

> 设计建议：识别号码后建议用「号码球」可视化让用户核对，而非纯文本输入框；并强化「确认无误」这一步的可信感。这是体现 AI 识别可纠错的关键环节。

---

### 5.3 彩票列表

**用途**：列出所有已保存的彩票，点击进入详情。

**内容元素**：列表，每行：
- 主标题：`[彩种] 第 X 期`
- 副标题（次要色）：最近一次结果，如「最近：中奖 ¥X」/「最近：未中奖/待确认」/「未验奖」。

**交互**：点行 → 进入「彩票详情」。

**状态**：空列表需空状态。

---

### 5.4 彩票详情（二级页）

**用途**：单张彩票的全部信息、换源重验、查看历史验奖记录。

**内容元素**
1. 彩票原图（若有），最大高度约 200pt。
2. 本票的注号（号码球：前区红 + 后区蓝），支持多注（未来复式）。
3. **再次验奖** 区：一排按钮，每个对应一个可用数据源（点击即用该源重新拉取并判奖，会**追加**一条记录而非覆盖）。
4. 状态提示文字。
5. **验奖记录**列表（按时间倒序），每条记录：
   - 数据源标签（胶囊样式）。
   - 版本号 `vN`。
   - 右侧结果：金额 / 「中奖(金额以官方为准)」/「未中奖」（中奖用绿色）。
   - 每注的中奖等级（或「未中奖」）。
   - 「查看/管理该期开奖版本」按钮 → 打开开奖版本弹窗。

**设计要点**：一张票可被多次、多源验奖，产生**多条历史记录**。需要清楚表达「每条记录来自哪个源 / 哪个版本 / 什么时候 / 结果如何」，并体现「记录不可被覆盖，只会追加」的可信赖感。

---

### 5.5 开奖版本管理（模态弹窗 Sheet）

**用途**：查看某一期某一来源的所有开奖号码「版本」，并可手动新增/修正版本、用指定版本重新判奖。固定宽度约 460pt。

**背景概念**：开奖结果是**不可变的版本快照**。同一期同一源可能有多个版本（如先抓取、后手动修正），编辑不会改旧版本，只会新增递增版本号。

**内容元素**
1. 标题：`开奖版本 · 彩种 第 X 期 · 来源`。
2. **版本列表**（按版本号倒序），每个版本：
   - `vN` + 来源类型标签（「抓取」/「手动」）。
   - 「用此版本重验」按钮。
   - 开奖号码球（前区红 + 后区蓝）。
   - 若有来源页链接，显示「来源页」可点链接。
3. 分割线 + **手动新增/修改版本** 区：
   - 前区/红球输入。
   - 后区/蓝球输入。
   - 错误提示（红色）。
4. 底部：「保存为新版本」 + 「关闭」。

**状态**：号码校验失败的错误提示。

---

### 5.6 验奖结果总览

**用途**：表格形式汇总每张彩票的**最新**验奖结果，可按彩种筛选。

**内容元素**
1. 顶部分段控件筛选：全部 / 双色球 / 大乐透。
2. **表格**，列：
   - 彩种
   - 期数
   - 最新结果（「中奖 ¥X」/「中奖(待定金额)」/「未中奖」/「未验奖」；中奖绿色）
   - 来源（数据源显示名）

**状态**：空数据状态。设计师可考虑响应式列宽、排序、斑马纹等表格规范。

---

### 5.7 统计

**用途**：图表化分析购彩与中奖情况。

**内容元素**
1. 顶部 4 张核心数据卡片（同首页：累计投入 / 累计中奖 / 净盈亏 / 中奖率）。
2. **彩种占比**：环形（甜甜圈）饼图。
3. **按日购买量**：柱状图（X=日期，Y=张数）。
4. **我的常选红球频率（双色球）**：柱状图（X=号码，Y=被选次数）。

**设计要点**：图表配色需与号码球红/蓝体系协调；给出无数据时的图表空状态。可建议增加时间范围筛选等。

---

### 5.8 设置

**用途**：配置 AI 识别模型与自建 Web 服务数据源。表单式（分区块）。

**内容元素**
1. **视觉模型（OpenAI 兼容）** 区：
   - Base URL（如 https://api.openai.com/v1）
   - API Key（密文输入 SecureField）
   - 模型名（如 gpt-4o）
2. **Web 服务数据源** 区：
   - 开关：启用 Web 服务数据源
   - Base URL（如 http://host:8080）
   - 共享 Token（密文输入）
3. 「保存」按钮 + 保存成功提示（绿色「已保存」）。

**设计要点**：敏感字段（Key/Token）密文展示；建议加字段说明/校验/连接测试反馈。未配置模型会直接影响「验奖」页能否识别——可在此处与验奖页之间建立引导关系。

## 7. 全局状态规范（每页都需考虑）

请为以下通用状态给出设计：
- **空状态**：首次使用、无彩票、无记录、图表无数据。
- **加载中**：识别中、验奖中、拉取开奖中。
- **错误**：识别失败、未配置模型、该期未开奖、网络/数据源失败、号码校验不通过。
- **成功**：验奖中奖（强反馈，建议有正向情绪表达）、保存成功。

> 注意：官方开奖接口存在反爬 / 地域限制，**线上可能失败**，「手动录入」是永远可用的兜底。错误态与「转手动」的引导很重要。
