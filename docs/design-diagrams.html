3. ER 数据模型 (App 本地)

以彩票为中心：一张票多条验奖记录；开奖按 (彩种+期数+源) 唯一并带不可变版本；验奖记录精确引用某版本。

erDiagram
  TICKET ||--o{ VERIFICATION_RECORD : "一张票多条验奖"
  DRAW ||--o{ DRAW_VERSION : "多个不可变版本"
  DRAW_VERSION ||--o{ VERIFICATION_RECORD : "被引用(可复现)"

  TICKET {
    uuid id
    string category "dlt|ssq"
    string issue
    json bets "确认后投注号码"
    string imageFileName "原图(沙箱文件)"
    decimal cost "默认 注数x¥2"
    date purchaseDate
    datetime createdAt
  }
  VERIFICATION_RECORD {
    uuid id
    ref ticket
    ref drawVersion "决定 源+版本+号码"
    json result "命中/奖级/金额/合计 快照"
    datetime createdAt
  }
  DRAW {
    uuid id
    string category
    string issue
    string source "体彩|福彩|webService|manual"
  }
  DRAW_VERSION {
    uuid id
    ref draw
    int versionNumber "1,2,3..."
    json frontNumbers
    json backNumbers
    json prizes "各奖级金额,可空"
    date drawDate
    string origin "fetched|manual"
    string sourceURL "仅fetched,可点击"
    datetime createdAt
  }

唯一键：DRAW = category + issue + source。版本不可变，手动修改 = 新增版本，旧版本与依赖它的旧验奖记录不受影响。

5. 时序图 · 首次验奖

识别 → 可编辑确认 → 保存彩票 → 选源(缓存优先)取版本 → 比对 → 保存验奖记录。

sequenceDiagram
  autonumber
  actor U as 用户
  participant UI as 界面
  participant VR as VisionRecognizer
  participant LLM as 视觉模型
  participant DS as DrawDataSource
  participant NET as 数据源(官方/Web)
  participant PE as PrizeEvaluator
  participant ST as Store

  U->>UI: 拖入彩票图片
  UI->>VR: recognize(image)
  VR->>LLM: chat/completions (图片+提示)
  LLM-->>VR: JSON(彩种/期数/号码)
  VR-->>UI: RecognizedTicket
  UI-->>U: 可编辑确认表单
  U->>UI: 核对/修改 + 选数据源 + 点验奖
  UI->>ST: 保存 Ticket
  UI->>DS: fetchDraw(category, issue, source)
  alt 该源缓存命中
    DS->>ST: 取最新 DrawVersion
  else 未命中
    DS->>NET: 请求开奖
    NET-->>DS: 号码/奖金/sourceURL
    DS->>ST: 建 Draw + DrawVersion(v1, fetched)
  end
  DS-->>UI: DrawVersion
  UI->>PE: evaluate(bets, drawVersion)
  PE-->>UI: 每注奖级/金额/合计
  UI->>ST: 保存 VerificationRecord(引用该版本)
  UI-->>U: 结果页

6. 时序图 · 再次验奖（换源 / 刷新）

同一张票，换数据源或刷新某源，追加一条验奖记录，便于跨源对比。

sequenceDiagram
  autonumber
  actor U as 用户
  participant UI as 彩票详情页
  participant DS as DrawDataSource
  participant NET as 另一数据源
  participant PE as PrizeEvaluator
  participant ST as Store

  U->>UI: 选另一数据源 / 刷新某源
  UI->>DS: fetchDraw(category, issue, source2)
  alt 命中且无需刷新
    DS->>ST: 取最新 DrawVersion
  else 刷新或未命中
    DS->>NET: 请求开奖
    NET-->>DS: 号码/sourceURL
    DS->>ST: 号码有变则新增 DrawVersion
  end
  DS-->>UI: DrawVersion
  UI->>PE: evaluate(bets, drawVersion)
  PE-->>UI: 结果
  UI->>ST: 追加 VerificationRecord
  UI-->>U: 该票多条验奖并存

7. 时序图 · 手动修改开奖版本

编辑即新增版本（origin=manual），历史版本与旧验奖记录稳定不变；可选新版本重验。

sequenceDiagram
  autonumber
  actor U as 用户
  participant SH as 开奖版本浮层
  participant ST as Store
  participant PE as PrizeEvaluator

  U->>SH: 打开某 Draw(彩种+期数+源)
  SH->>ST: 读取版本历史
  ST-->>SH: v1(fetched) ... vN
  U->>SH: 编辑号码/奖金
  SH->>ST: 新增 DrawVersion(vN+1, manual)
  Note over ST: 旧版本不可变
旧验奖记录仍指旧版本
  U->>SH: 选新版本重验
  SH->>PE: evaluate(bets, 新版本)
  PE-->>SH: 结果
  SH->>ST: 追加 VerificationRecord(引用新版本)

8. 时序图 · Web 服务录入 + App 拉取

浏览器登录后录入开奖（号码+可选奖金）入库；App 经 API 拉取并存为 source=webService 的版本。

sequenceDiagram
  autonumber
  actor A as 我(浏览器)
  participant FE as React 前端
  participant API as FastAPI
  participant DB as 数据库
  participant APP as Mac App

  A->>FE: 登录(共享口令)
  FE->>API: POST /api/v1/auth/login
  API-->>FE: Token
  A->>FE: 录入 期数+号码(+奖金)
  FE->>API: POST /api/v1/draws (Bearer)
  API->>DB: upsert (category+issue)
  DB-->>API: ok
  Note over APP: 之后 App 验奖选用 Web 服务源
  APP->>API: GET /api/v1/draws/{cat}/{issue} (Bearer)
  API->>DB: 查询
  DB-->>API: 开奖记录
  API-->>APP: JSON(号码/奖金?)
  APP->>APP: 存为 DrawVersion(source=webService, fetched)
