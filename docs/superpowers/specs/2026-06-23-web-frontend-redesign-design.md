# Web 前端重设计 + Mac App 介绍首页 — 设计文档

- **日期**:2026-06-23
- **范围**:重新设计 `webservice/frontend/` 现有 4 个页面,新增 1 个公开首页(介绍 Mac App)
- **交付**:先用 Claude Design 出高保真 HTML 原型 → 确认后落地到现有 React + Vite 工程
- **样式方案**:原生 CSS + 设计令牌(CSS Variables)+ CSS Modules,零额外依赖

## 1. 目标与背景

现有 Web 前端(React 18 + Vite + react-router 6)是功能性 MVP,**完全没有样式**(纯 HTML 元素),包含 4 个页面:登录、开奖列表、录入表单、开奖详情,用于手动管理大乐透 / 双色球开奖号码。

本次目标:
1. 为 4 个现有页面做完整视觉重设计。
2. 新增一个面向公众的**首页 / Landing**,介绍配套 Mac App(拍照识别 → 验奖 → 记录)。
3. 整站建立统一的苹果风设计系统。

**视觉方向**:苹果风简约打底(大量留白、圆角卡片、柔和阴影、系统字体)+ 彩色号码球点缀(双色球红蓝、大乐透红金)+ 数据专业感(整齐的数据网格、等宽数字)。

## 2. 信息架构与路由

首页公开;数据工具页迁移到 `/draws/*` 子路径并保留登录。

| 路径 | 页面 | 权限 | 备注 |
|------|------|------|------|
| `/` | 首页 / Landing | 公开 | 全新页面,介绍 Mac App |
| `/login` | 登录 | 公开 | 重设计 |
| `/draws` | 开奖列表 | 需登录 | 原 `/` 迁移 |
| `/draws/new` | 录入表单 | 需登录 | 原 `/new` |
| `/draws/edit/:category/:issue` | 编辑表单 | 需登录 | 原 `/edit/:category/:issue` |
| `/draws/:category/:issue` | 开奖详情 | 需登录 | 原 `/draw/:category/:issue` |

**路由迁移注意**:`App.tsx` 中现有 `/`、`/new`、`/edit/...`、`/draw/...` 全部改到 `/draws` 前缀;`/` 改为公开 Landing。`RequireAuth` 仅包裹 `/draws/*`。登录成功后回跳到来源页(若直接访问受保护页,登录后回到该页;否则回 `/draws`)。

**全局导航**:
- 公开页(首页 / 登录)使用轻量顶栏:左侧 Logo「验奖」,右侧「进入数据管理」按钮 → `/draws`(未登录先到 `/login`,登录后回跳)。
- 工具页(`/draws/*`)使用统一 App 顶栏:Logo + 「开奖记录」+ 退出登录。
- 抽出共享 `Layout`(或 `PublicLayout` / `AppLayout` 两种)组件,避免每页重复顶栏。

## 3. 设计系统(设计令牌 + 可复用元素)

落地为一份 `tokens.css`(`:root` 变量)+ 若干可复用组件 + 每个组件的 CSS Module。

### 3.1 颜色令牌

```
--bg:              #FBFBFD   /* 页面底色,近白 */
--surface:         #FFFFFF   /* 卡片 */
--border:          #E5E5EA   /* 浅灰描边 */
--text:            #1D1D1F   /* 主文字,近黑 */
--text-secondary:  #6E6E73   /* 次要文字,灰 */
--accent:          #0071E3   /* 苹果蓝,按钮/链接/CTA */

/* 彩种品牌色 — 用于号码球与标签 */
--ssq-red:         #E63946   /* 双色球红球 */
--ssq-blue:        #1D6FB8   /* 双色球蓝球 */
--dlt-front:       #E63946   /* 大乐透前区 */
--dlt-back:        #F4A300   /* 大乐透后区,金 */

/* 状态色 */
--win-green:       #34C759   /* 中奖 */
/* 未中奖使用 --text-secondary */
```

### 3.2 字体 / 圆角 / 阴影 / 间距

- **字体栈**:`-apple-system, "SF Pro Text", "PingFang SC", system-ui, sans-serif`(中文用苹方)。数字密集处(奖金表、号码)用等宽数字特性 `font-variant-numeric: tabular-nums`。
- **圆角**:卡片 `16px`、按钮 `12px`、号码球 `999px`。
- **阴影**:`--shadow-card: 0 4px 20px rgba(0,0,0,.06)`(柔和低对比)。
- **间距刻度**:4 的倍数 — 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64。

### 3.3 可复用组件

- **`NumberBall`** — 彩色号码球(核心视觉元素)。props:`value: number`、`type: 'ssq-red' | 'ssq-blue' | 'dlt-front' | 'dlt-back'`、`size: 'lg' | 'md' | 'sm'`。圆球 + 居中数字 + 细微高光。
- **`Card`** — 白底圆角柔阴影容器,贯穿列表 / 详情 / 表单。
- **`Button`** — 三档:`primary`(苹果蓝实心)/ `secondary`(灰描边)/ `text`(纯文字)。支持禁用态(用于「敬请期待」CTA)。
- **`Badge`** — 彩种标签(双色球红、大乐透金)与中奖状态标签(中奖绿 / 未中灰)。
- **`DataField`** — 详情页「标签 + 值」成对展示,整齐网格 + 等宽数字(数据专业感)。
- **`Segmented`** — 苹果风分段控件,用于彩种筛选 / 表单彩种选择(全部 / 双色球 / 大乐透)。

## 4. 页面布局

所有页面响应式:桌面多列、移动端单列堆叠。

### 4.1 首页 / Landing(`/`,公开,全新)

苹果产品页式纵向分段滚动:

1. **Hero**:大标题「拍张照,自动验奖」+ 一句话副标题(价值主张)+ Mac App 截图占位框(大圆角 + 柔阴影,居中或右侧)+ 主 CTA「下载 App · 敬请期待」(禁用态浅色)。
2. **核心功能区**:5 个功能卡片(图标占位 + 标题 + 一句话),覆盖全部卖点:
   - 拍照识别号码(多模态 AI)
   - 一键验奖(大乐透 / 双色球)
   - 自动获取官方开奖结果
   - 中奖记录与盈亏图表
   - 数据本地存储 / 隐私
   桌面 3+2 或 2 列网格,移动端单列。
3. **流程区**:「拍照 → 识别确认 → 验奖 → 记录」四步,配号码球点缀的小示意图(占位)。
4. **数据感小节**:「支持 大乐透 / 双色球」双标签 + 一条号码球展示带,呼应专业感。
5. **页脚**:项目名、GitHub 链接、「进入数据管理」入口。

**素材说明**:暂无 App 截图 / 图标 / 下载链接,全部用占位框 + 禁用态 CTA,后续替换。

### 4.2 登录(`/login`,公开,重设计)

居中单卡片:Logo + 「数据管理登录」标题 + 密码输入框(后端是共享密码 → Bearer token)+ 主按钮 + 错误提示(红色内联)。背景 `--bg`,卡片柔阴影。极简。

### 4.3 开奖列表(`/draws`,重设计)

- **顶部**:标题「开奖记录」+ 彩种筛选 `Segmented`(全部 / 双色球 / 大乐透)+ 右侧「+ 录入」主按钮。
- **主体**:卡片列表,每条一张 `Card` — 左侧彩种 `Badge` + 期号(+ 日期),中间一排 `NumberBall`(前区红 + 后区蓝/金),右侧箭头进详情。
- **空态**:占位提示 + 「+ 录入」引导。
- **错误态**:顶部内联 `role="alert"` 提示(保留现有错误处理语义)。

### 4.4 录入 / 编辑表单(`/draws/new`、`/draws/edit/:category/:issue`,重设计)

单列卡片表单,分组:

- **基本信息**:彩种 `Segmented`、期号、开奖日期。
- **开奖号码**:一排"球形输入位"——点击位填数字,实时渲染成对应颜色的 `NumberBall`(所见即所得)。前区 / 后区分组,数量按彩种规则(大乐透前 5 后 2,双色球前 6 后 1)。
- **奖金(可选)**:一 / 二等奖金额输入。
- **底部**:保存(primary)/ 取消(secondary)。校验错误内联红字(复用现有 `validation.ts` 规则)。

### 4.5 开奖详情(`/draws/:category/:issue`,重设计)

- **顶部**:彩种 `Badge` + 期号 + 日期。
- **开奖号码**:大号 `NumberBall` 居中展示(全站视觉高潮)。
- **奖级 / 奖金表**:`DataField` 网格或表格,列出各奖级与金额,等宽数字对齐(数据专业感)。
- **底部**:编辑 / 返回列表。

## 5. 技术落地

- **样式**:新建 `src/styles/tokens.css`(全局令牌,`main.tsx` 引入)。组件与页面各自 `*.module.css`。
- **组件目录**:`src/components/`(`NumberBall`、`Card`、`Button`、`Badge`、`DataField`、`Segmented`、`PublicLayout`、`AppLayout`)。
- **页面目录**:`src/pages/` 现有 4 页重写 + 新增 `LandingPage.tsx`。
- **路由**:改写 `App.tsx`(见第 2 节);登录回跳逻辑。
- **不改动**:`api.ts`、`auth.ts`、后端、`validation.ts` 的校验逻辑(仅复用)。
- **测试**:保留并按需调整现有 `DrawFormPage.test.tsx`、`validation.test.ts`;路由变更后更新链接断言。

## 6. 工作流程

1. 用 Claude Design 基于本设计出 5 个页面的高保真 HTML 原型,渲染预览。
2. 用户确认原型效果。
3. 落地到 React:建立设计令牌与组件 → 重写 / 新增页面 → 调整路由 → 跑通 `npm run dev` 与 `npm test`。

## 7. 非目标(YAGNI)

- 不引入 Tailwind / 组件库等额外依赖。
- 不改动后端 API 与数据契约。
- 暂不做真实 App 截图 / 下载链接(占位,后续替换)。
- 暂不做暗色模式、国际化、动效系统(可后续迭代)。
